# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

Naxus takes the security of Unicorn HRMS seriously. If you discover a security vulnerability, please follow these steps:

### Reporting Process

1. **Do Not** disclose the vulnerability publicly
2. Send details to: security@naxus.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

### Response Timeline

- **Initial Response:** Within 48 hours
- **Status Update:** Within 7 days
- **Fix Timeline:** Varies based on severity

### Security Measures

Unicorn HRMS implements:
- JWT-based authentication
- Role-based access control (RBAC)
- Data encryption at rest and in transit
- Regular security audits
- Automated vulnerability scanning
- Comprehensive audit logging

## Responsible Disclosure

We appreciate responsible disclosure and will:
- Acknowledge your contribution
- Keep you informed of our progress
- Credit you in release notes (unless you prefer anonymity)

## Security Best Practices

For system administrators:
- Keep systems updated
- Use strong passwords
- Enable two-factor authentication
- Regular backup procedures
- Monitor audit logs
- Restrict network access

---

For urgent security matters, contact: security@naxus.com
